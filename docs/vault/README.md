# LS-Spline Dissertation Vault

This is your Obsidian knowledge base for the dissertation:
**"Bayesian Spatial Additive Regression with Lancaster–Šalkauskas Splines and Matérn Gaussian Process Random Effects"**

## How to open this in Obsidian

1. Download Obsidian from https://obsidian.md (free for personal use)
2. Open Obsidian → "Open folder as vault" → select this folder
3. Open `00 Maps of Content/MOC - Dissertation.md` — that's your home base

## How this vault is organized

The folders are loose organization. The real structure comes from `[[wiki-links]]` between notes.

- **00 Maps of Content** — hand-curated index notes. Start here.
- **10 Methods** — atomic notes, one concept per note (LS basis, RW2 prior, etc.)
- **20 Simulations** — one note per experiment, with settings + results + status
- **30 Applications** — real-data analyses (Munich rent, SEER queued)
- **40 Literature** — one note per key paper, with how it relates to your work
- **50 Decisions and Insights** — *why* you made design choices. The most valuable notes.
- **60 Code and HPC** — file index, Hellbender notes, bug log
- **70 Daily** — daily working notes (use as a scratchpad)
- **80 Templates** — copy-paste templates for new notes

## Recommended Obsidian settings to enable

**Core plugins** (Settings → Core plugins):
- Daily notes
- Backlinks
- Outgoing links
- Graph view
- Templates → set folder to `80 Templates`
- Outline

**Community plugins to consider** (Settings → Community plugins → Browse):
- **Dataview** — query notes like a database (e.g., list all simulations with their RMSE)
- **Excalidraw** — hand-drawn diagrams of model structure
- **Iconize** — folder icons

## How to use this day-to-day

1. **Start each day** by creating a daily note (`70 Daily/YYYY-MM-DD.md`). Jot what you're working on.
2. **When a concept comes up**, type `[[Concept Name]]` — if the note doesn't exist, click to create it.
3. **When you decide something or learn something**, write a note in `50 Decisions and Insights`. These are the highest-value notes you'll ever write.
4. **When you finish a simulation**, copy `80 Templates/Simulation Template.md` into `20 Simulations/` and fill it in *while it's fresh*.
5. **Check the backlinks panel** on any note — it shows everywhere that note is referenced. This is how connections emerge.

## Don't try to migrate everything at once

These seed notes are deliberately incomplete. Many notes contain `[fill in]` placeholders. That's the point — review, edit, and add as you go. Treat this as a living document, not a finished product.
